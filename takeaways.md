## Practical takeaways for astronomers

### A minimum reproducibility bar for ML projects
Encourage learners to aim for:
- Versioned code
- Fixed random seeds (documented)
- Explicit data splits
- Environment description
- Clear statement of limitations

### When full openness isn’t possible
- You can still:
  - describe data access conditions
  - document preprocessing
  - share model cards or summaries
  - state intended use explicitly

### Language to encourage
- “Exploratory result”
- “Model valid within this domain”
- “Performance not tested beyond…”

### Role of ADACS
- Training and best practice guidance
- Support for sustainable software
- Community norms for ML use


> Good ML ethics is mostly good scientific hygiene — ML just raises the stakes.
